# Plan: Allow offscreen HMI elements in editor, filter them in runtime

## Summary
Remove editor constraints on HMI element positioning (allow negative & beyond-screen coordinates).
Add runtime filtering so completely-offscreen objects are not rendered and not subscribed to tags.
Editor mode shows all objects regardless of position.

- Export `intersectsScreenBounds(object: { x, y, width, height }, screen: { width, height }): boolean`
- Returns true if object's bounding box intersects screen rect `{x:0, y:0, width: screen.width, height: screen.height}`
- Implementation: standard AABB intersection test

## Detailed changes

### A. `offscreen-filter.ts` (new file)
```typescript
export function intersectsScreenBounds(
  object: { x: number; y: number; width: number; height: number },
  screen: { width: number; height: number }
): boolean {
  const ox = object.x ?? 0;
  const oy = object.y ?? 0;
  const ow = Math.max(0, object.width ?? 0);
  const oh = Math.max(0, object.height ?? 0);
  
  return !(
    ox + ow < 0 ||
    oy + oh < 0 ||
    ox > screen.width ||
    oy > screen.height
  );
}
```

### B. Editor scroll clamping removal

## Known risks
- Editor Stage expansion with GROUP offset is the most invasive change; careful testing needed for:
  - Zoom centering
  - Scroll position after screen switch
  - Selection rect coordinates
  - Transformer position
  - Pan tool
- Runtime tag subscription filtering must not break frame/libraryElementInstance nested tag collection
- `ObjectCommunicationOverlayNode` uses `collectRuntimeObjectResolvedTags` which iterates object fields; the function is called per-object so offscreen filtering at parent level prevents it from running
